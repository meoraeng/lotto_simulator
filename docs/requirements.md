# 요구사항 정리
> 기존 우테코 프리코스 과제에 명시되었던 학습목표, 요구사항, 공통 피드백 등을 Go에 맞춰 정리



## 유지
Go 환경에서도 그대로 유지할 기존 요구사항 리스트

- 로또 기존 요구사항
- 단위 테스트 적용
- 성공&예외 모두 테스트 작성
- 예외 발생 시 의미있는 예외로 분류
- 함수 짧게 유지
- 책임 분리
- else 사용하지 않기
    - switch 쓰지 않기
- Git 컨벤션
- 에러 출력 시 PREFIX 
- 하드코딩보다는 상수 지향
- getter/setter 지양
- 3항 연산자 쓰지 않기
- 잘못된 입력에 대해 재입력 받기

## 수정
Go 환경에 맞춰 수정한 요구사항

- enum 사용 -> Go iota로 대체
- try-catch -> error 반환값 사용
- 객체 협력 -> struct와 함수로 대체
- Java API 사용 -> Go 표준 라이브러리
- 로그의 성격을 띄는 객체 출력 toString -> 구조체의 출력 fmt.Stringer 인터페이스 구현
