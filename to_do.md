### 기능 리스트 정리 
- [X] 구매 금액대로 로또 랜덤 생성하여 저장
- [X] 생성된 로또들 콜렉션으로 관리
    - 당첨 번호와 보너스 번호 입력
- [X] 안내멘트 출력 후 입력받기(금액, 당첨번호, 보너스번호)
- [X] 공통 - 빈 문자열인지 검사
- [X] 공통 - 숫자 입력이 맞는지 검사
- [X] 금액 - 1000원으로 나누어 떨어지는지 검사 후 파싱
- [X] 당첨번호 - 구분자가 정상적인지 검사 -> 구분자 기준으로 분리
- [X] 당첨번호/보너스번호 - 범위 내의 값인지 검사(1~45)
- [X] 당첨번호/보너스번호 - 중복되지 않았는지 검사
- [X] 당첨번호/보너스번호 - 파싱
- [X] 로또 당첨번호 저장
- [X] 로또 보너스 번호 저장
- [X] 로또 등수 조건 정의
- [X] 랜덤생성 로또와 당첨번호&보너스번호 대조하여 등수 결과 카운트(총합 계산) 당첨 결과와 수익률 출력
- [X] 랜덤생성된 로또 결과 출력(총 수량, 각 번호)
- [X] 당첨 결과 출력
- [X] 총 수익률 출력
### 추가 기능 구현 리스트 
- [X] 모드(고정/분배) 전환을 위한 라운드 구조 작성
- [X] 분배 모드 분배 로직 구현
    - 등수별 계산
    - 이월 금액을 합산한 뒤, 당첨자 수에 따라 균등 분배 or 전액 이월 처리
- [X] 라운드 잔액 처리 규칙 정의
    - 1~3자리수 단위에 대한 정책 결정
- [X] 등수별 상한 관련 규칙 구현
    - 롤다운 : 상한 초과분은 하위 등수로 내려보낸다
    - 이월: 하위 등수에도 당처자가 없을 경우 초과분 이월 처리
- [X] N회 시뮬레이션 구현
    - N회 연속으로 시뮬레이션 돌리고 등수별 상태를 회차 간 유지
    - 최종 회차별 통계 요약한 리포트 생성
- [X] 모드 전환 및 설정 주입
- [X] 플레이어/티켓 레이어 도입
    - 사람/티켓 입력 받아서 Winners 집계 -> RO 기반 per-player 지급 계산
- [X] 테스트 케이스 확장
    - 추가된 기능에 맞는 테스트 추가
- [X] 플랫폼 확장
 - CLI 외에 HTTP API 환경에서도 결과를 확인할 수 있도록
